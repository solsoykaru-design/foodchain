🔴 Критические ошибки (10)

К-1 CriticalАутентификация
Хардкодный бэкдор admin:admin — Любой, кто знает admin:admin, получает суперадминский JWT. Аккаунт создаётся на лету.
server/routes/auth.js:293
✅ Удалить блок if (username === 'admin' && password === 'admin'). Использовать только нормальную аутентификацию через БД.
К-2 CriticalАутентификация
Хардкодный JWT_SECRET fallback — Если JWT_SECRET не задан в env, используется 'foodchain-staff-secret'. Любой может подделать токен.
server/index.js:33
✅ Убрать fallback. process.exit(1) если секрет не задан в любом режиме, а не только в production.
К-3 CriticalМультитенантность
tenant_id из query-параметра — Неавторизованные запросы могут указать ?tenant_id=5 и получить доступ к данным любого арендатора.
server/index.js:441-445
✅ Никогда не брать tenant_id из query-параметров. Использовать только из JWT. Если токена нет — вернуть 401.
К-4 CriticalБезопасность
Реальные секреты в .env закоммичены в git — JWT_SECRET, TELEGRAM_BOT_TOKEN, SUPABASE_SERVICE_ROLE_KEY, PAYME_MERCHANT_ID, OPENCODE_API_KEY — все в открытом виде.
.env, portal/backend/.env, server/portal-backend/.env
✅ Немедленно удалить из git-истории (BFG Repo-Cleaner). Сбросить все скомпрометированные ключи. Добавить .env в .gitignore.
К-5 CriticalBackend
Нет глобального обработчика ошибок Express — При любой синхронной ошибке сервер возвращает HTML-страницу со стектрейсом или падает.
server/index.js — отсутствует app.use((err, req, res, next) => {...})
✅ Добавить 4-параметровый error handler, возвращающий JSON { error: '...' } с safeError().
К-6 CriticalBackend
7+ роутов без try/catch — GET /api/orders, GET /api/orders/track, POST /api/couriers, GET /api/notifications и другие не имеют обработки ошибок. При ошибке SQL сервер подвисает.
routes/orders.js:5,23,35; routes/misc.js:48,55,64
✅ Обернуть все route handlers в try/catch с возвратом 500 + JSON.
К-7 CriticalМультитенантность
SQL OR-приоритет ломает tenant-изоляцию — Авто-инжекция AND tenant_id = current_tenant_id() добавляется БЕЗ скобок. WHERE a OR b → WHERE a OR (b AND tenant_id) — данные из других tenant'ов утекают через условие ДО OR.
server/index.js:383-388
✅ Добавлять AND tenant_id с правильными скобками: оборачивать оригинальный WHERE в (...) или переписать парсер SQL.
К-8 CriticalМультитенантность
40+ эндпоинтов используют req.query.tenant_id вместо req.tenant_id — Во всех админ-роутах, телеграм-роутах, misc-роутах tenant_id берётся из URL-параметра, а не из JWT.
routes/admin.js (строки 40,46,70,77,84,90…), routes/telegram.js, routes/misc.js, routes/staff.js
✅ Заменить все req.query.tenant_id || 1 на req.tenant_id. Если req.tenant_id не задан — вернуть 401.
К-9 CriticalИнфраструктура
Хардкодный Render URL в десктоп-админке и мобильном приложении — https://foodchain-qpxh.onrender.com зашит в коде. При смене домена всё ломается.
desktop-admin/preload.js:3, mobile/services/auth.tsx:4
✅ Использовать переменные окружения (process.env.API_URL, import.meta.env.VITE_API_URL).
К-10 CriticalМультитенантность
Mobile.js — нулевая tenant-изоляция — Весь модуль мобильных пользователей (mobile_users, mobile_payments, dish_catalog) не использует tenant_id вообще. Все данные всех арендаторов перемешаны.
server/routes/mobile.js — весь файл
✅ Добавить колонку tenant_id во все таблицы mobile-модуля. Добавить фильтр по tenant_id во все запросы.

🟠 Высокий приоритет (15)

В-1 HighАутентификация
JWT verify без указания алгоритма — Все 16 вызовов jwt.verify() не указывают { algorithms: ['HS256'] }. Уязвимость algorithm confusion (CVE-2015-9235).
Все jwt.verify() вызовы: index.js, middleware/auth.js, routes/auth.js, portal-backend
✅ Добавить { algorithms: ['HS256'] } во все jwt.verify().
В-2 HighAPI
Нет валидации данных — Ни один route handler не использует Zod/Joi/express-validator. Все проверки ручные, неполные, неконсистентные.
Все route-файлы
✅ Внедрить Zod-схемы для всех входных данных (body, query, params).
В-3 HighБезопасность
Пароль суперадмина выводится в console.log — При старте сервер печатает пароль в stdout, который попадает в логи Render.
server/index.js:4474,4486
✅ Убрать console.log('SUPERADMIN PASSWORD: ' + password). Выводить только логин.
В-4 HighБезопасность
Пароль в email в открытом виде — Welcome-email содержит пароль в HTML-теле. Пароль передаётся по сети и сохраняется в почтовых архивах.
server/routes/misc.js:765-779
✅ Отправлять одноразовую ссылку для установки пароля вместо пароля в теле письма.
В-5 HighБезопасность
Загрузка файлов без fileFilter — Chat, branding, site-settings uploads не проверяют MIME-тип. Можно загрузить .exe, .php, .html.
server/index.js:3372-3378, routes/branding.js:50-62
✅ Добавить fileFilter с проверкой MIME + magic bytes для всех upload-роутов.
В-6 HighИнфраструктура
localtunnel в production dependencies — Утилита для отладки работает в production. Добавляет поверхность атаки и лишние зависимости.
server/package.json:22
✅ Переместить в devDependencies.
В-7 HighИнфраструктура
Puppeteer в production dependencies — Скачивает Chromium (~300MB) на сервер. Увеличивает время деплоя и расход памяти.
server/package.json:24
✅ Переместить в devDependencies или использовать puppeteer-core.
В-8 HighИнфраструктура
dist-* папки закоммичены в git — Сборки фронтенда не должны быть в репозитории. Раздувают историю, создают конфликты merge.
dist-admin/, dist-waiter/, dist-guest/ и др. в корне
✅ Добавить dist-* в .gitignore. Собирать на CI/CD или сервере.
В-9 HighАрхитектура
Дублирование portal-backend — Полная копия portal/backend в server/portal-backend/. Разные версии могут рассинхронизироваться.
portal/backend/ vs server/portal-backend/
✅ server/portal-backend/ нужен для Render из-за особенностей деплоя, но должен быть симлинком или авто-копией при деплое.
В-10 HighАутентификация
/api/mobile/admin/set-unlimited без аутентификации — Любой может выдать безлимитную подписку любому номеру телефона.
server/routes/mobile.js:775-783
✅ Добавить проверку JWT + роли admin.
В-11 HighАутентификация
/api/backup без аутентификации — Любой может инициировать бэкап БД.
server/index.js:4456-4458
✅ Добавить аутентификацию + проверку роли.
В-12 HighМультитенантность
getOrderFull() не фильтрует по tenant_id — Функция SELECT * FROM orders WHERE id = ? вызывается из неаутентифицированных эндпоинтов. Любой заказ по ID доступен любому.
server/index.js:586-596
✅ Добавить AND tenant_id = current_tenant_id() во все запросы внутри getOrderFull().
В-13 HighБезопасность
Нет CSRF-защиты — Ни один пакет для защиты от CSRF не используется. CORS настроен на origin: true (любой источник).
server/index.js:106, весь код
✅ Добавить double-submit cookie pattern или SameSite=Strict. Ограничить CORS конкретными доменами.
В-14 HighБезопасность
50MB лимит тела запроса — express.json({ limit: '50mb' }) — чрезмерно большой лимит. Уязвимость для DoS-атак через большие payload.
server/index.js:148
✅ Снизить до 1-5 MB. Для файлов использовать multer с отдельными лимитами.
В-15 HighИнфраструктура
Нет HTTP→HTTPS редиректа, нет HSTS — Сервер слушает только HTTP. На Render HTTPS terminates на gateway, но если обойти — трафик незащищён.
Инфраструктура Render
✅ Настроить перенаправление HTTP→HTTPS на уровне Render или приложения. Включить HSTS.
