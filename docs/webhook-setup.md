# Настройка webhook-уведомлений для Т-Банка (Т-Касса)

## 1. Что это даёт

Webhook (он же Notification URL, Callback URL) позволяет Т-Банку автоматически уведомлять ваш сервер об изменении статуса платежа. Без webhook система не узнает, что клиент оплатил заказ, пока не произойдёт ручная проверка.

Вот что обрабатывается автоматически:

| Статус платежа | Действие системы |
|---|---|
| `CONFIRMED` (успешная оплата) | Заказ → «Оплачен», платеж → `succeeded`, уведомление админу |
| `CANCELED` / `REVERSED` (отмена/ошибка) | Заказ → «Ошибка оплаты», платеж → `failed`, уведомление админу |
| `REFUNDED` (возврат) | Заказ → снята отметка оплаты, платеж → `refunded` |
| Повторные уведомления | Игнорируются (если статус уже терминальный) |

## 2. Как получить URL для уведомлений

1. Зайдите в **админ-панель** → **Настройки** → **Платёжные системы**.
2. В карточке **«Т-Банк (Т-Касса)»** найдите поле **«URL для уведомлений (Notification URL)»**.
3. Поле автоматически заполняется на основе текущего домена:  
   `https://ваш-домен/api/webhooks/payment`
4. Нажмите **«Копировать»** — URL скопируется в буфер обмена.
5. Нажмите **«Сохранить и включить»**, чтобы URL сохранился в БД.

> Если ваш сервер работает на локальном IP, используйте `http://<ваш-внешний-IP>:4000/api/webhooks/payment`

## 3. Где указать URL в личном кабинете Т-Банка

1. Войдите в личный кабинет Т-Банка (Т-Касса):  
   - **Боевой режим:** https://securepay.tbank.ru  
   - **Тестовый режим:** https://rest-api-test.tbank.ru
2. Перейдите в раздел **«Магазин»** → выберите ваш терминал.
3. Вкладка **«Настройки»** → раздел **«Уведомления»**.
4. В поле **«Адрес для уведомлений»** (Notification URL) вставьте скопированный URL.
5. В поле **«IP-адреса для уведомлений»** можно указать IP вашего сервера (опционально).
6. Нажмите **«Сохранить»**.

## 4. Как это работает (техническая схема)

```
Клиент оплачивает заказ на сайте
        │
        ▼
Т-Банк принимает платёж
        │
        ▼
Т-Банк отправляет POST-запрос на → https://ваш-домен/api/webhooks/payment
        │
        ▼
Сервер проверяет подпись (Token):
  │  1. Исключает поле Token из тела запроса
  │  2. Сортирует поля по алфавиту
  │  3. Конкатенирует значения + Secret Key
  │  4. SHA-256 → compare с Token
  │                          │
            ┌─────────────────┐
            ▼                 ▼
        Подпись верна    Подпись не верна → 403 Forbidden
            │
            ▼
  ┌──────────────────────────────┐
  │ Проверка дубликата:          │
  │ статус уже succeeded/failed? │
  │ → ДА: игнорировать           │
  │ → НЕТ: обработать            │
  └──────────────────────────────┘
            │
            ▼
  Обновить status в payments + orders
  Отправить WebSocket-уведомление админу
  Записать в webhook_logs для аудита
```

## 5. Проверка подписи (алгоритм)

Сервер использует **три метода** проверки подписи (по приоритету):

1. **X-Signature header** — HMAC-SHA256 от тела запроса с Secret Key.  
2. **Token в теле запроса** — SHA-256 от конкатенации отсортированных значений параметров + Secret Key.  
   - Параметры сортируются по имени (ASCII), поле `Token` исключается.  
   - Строка: `value1 + value2 + ... + valueN + secretKey`.  
   - Результат: `SHA-256(string).toLowerCase()`.  
3. **Библиотека `tbank-payments`** — fallback, если первые два не сработали.

## 6. Логирование (аудит)

Все входящие webhook-запросы записываются в таблицу `webhook_logs`:

| Поле | Описание |
|---|---|
| `provider` | `tbank` (или другой провайдер) |
| `payment_id` | Идентификатор платежа от Т-Банка (`PaymentId`) |
| `payload` | Полный JSON тела запроса |
| `signature` | Значение `Token` или `X-Signature` |
| `status` | `processed` / `ignored` / `error` |
| `error_message` | Причина, если статус не `processed` |
| `processed_at` | Время обработки |

## 7. Тестирование

### Проверить, что webhook работает:

1. Включите **боевой режим** в админке (test_mode = 0).
2. Сделайте тестовый платёж на сайте.
3. Проверьте, что заказ перешёл в статус «Оплачен».
4. Проверьте `webhook_logs` в БД:  
   ```sql
   SELECT * FROM webhook_logs WHERE provider = 'tbank' ORDER BY processed_at DESC;
   ```

### Если webhook не приходит:

- Убедитесь, что сервер доступен из интернета (не заблокирован firewall, есть внешний IP/домен).
- Проверьте, что URL в личном кабинете Т-Банка указан без опечаток.
- Проверьте логи сервера: `PAYMENT_WEBHOOK_ERROR`.
- Проверьте `webhook_logs` — если записей нет, значит запрос не доходит до сервера.

## 8. Миграция БД

SQL-скрипт для добавления таблицы и колонок находится в:
```
server/migrations/006_webhook_support.sql
```

Приложение создаёт их автоматически при запуске (`initTables`), но для чистой установки можно выполнить скрипт вручную:
```bash
sqlite3 foodchain.db < server/migrations/006_webhook_support.sql
```
