# Приём онлайн-платежей через Т-Банк (Т-Касса)

Модуль интеграции с интернет-эквайрингом **Т-Банка** (бывший Тинькофф).  
Использует библиотеку [`tbank-payments`](https://github.com/esurkov1/tbank-payments) (v1.2.1).

## Настройка

### 1. Получить Terminal Key и Secret Key в личном кабинете Т-Банка

1. Войдите в [личный кабинет Т-Банка](https://www.tbank.ru/business/)
2. Перейдите в раздел **Эквайринг → Интернет-эквайринг**
3. Создайте терминал, если ещё не создан
4. В настройках терминала найдите:
   - **Terminal Key** — идентификатор терминала (например, `1700000000`)
   - **Secret Key** — пароль для API (например, `a1b2c3d4-e5f6-...`)
5. Скопируйте оба значения

### 2. Ввести ключи в админ-панели

1. Перейдите в **Настройки → Платёжные системы**
2. Найдите карточку **Т-Банк (Т-Касса)**
3. Заполните поля:
   - `Terminal Key` — скопированный ID терминала
   - `Secret Key` — скопированный пароль
4. Нажмите **«Проверить»**
   - Система создаст тестовый платёж на 1 копейку и сразу его отменит
   - При успехе: появится зелёная кнопка **«✅ Сохранить и включить»**
   - При ошибке: вы увидите сообщение с деталями
5. Нажмите **«✅ Сохранить и включить»** — данные сохранятся в БД, приём платежей активируется

### 3. Webhook URL

Для получения уведомлений о статусе платежа укажите в личном кабинете Т-Банка URL:

```
POST /api/webhooks/payment
```

Заголовок: `X-Provider: tbank`

### 4. URL успеха и ошибки

Укажите в настройках терминала Т-Банка:
- **Success URL** — страница, куда вернётся пользователь после успешной оплаты
- **Fail URL** — страница при ошибке оплаты

## API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/payment/create` | Создание платежа |
| `GET` | `/api/payment/status/:id` | Статус платежа |
| `POST` | `/api/payment/refund` | Возврат средств |
| `POST` | `/api/webhooks/payment` | Webhook от Т-Банка |
| `POST` | `/api/admin/payment/settings/:provider/test` | Проверка соединения |
| `PUT` | `/api/admin/payment/settings/:provider` | Сохранение настроек |

## Файлы модуля

- `server/payment/providers/tbank.js` — провайдер Т-Банка (использует `tbank-payments`)
- `server/payment/index.js` — общий модуль платежей (эндпоинты, webhook, подписки)
- `server/crypto-utils.js` — AES-256-GCM шифрование Secret Key
- `src/admin/PaymentSettingsPage.tsx` — страница настройки платёжных систем
- `src/admin/PaymentsPage.tsx` — список транзакций с возвратами и экспортом
- `src/guest/GuestApp.tsx` — гостевое приложение (CheckoutPage + PaymentProcessingPage)
- `src/api.ts` — API-функции клиента

## Безопасность

- **Secret Key шифруется** алгоритмом AES-256-GCM перед сохранением в БД
- Ключ шифрования — SHA-256 от `process.env.ENCRYPTION_KEY` (или дефолтный для разработки)
- Secret Key никогда не передаётся на клиент в открытом виде
- Все API-запросы к Т-Банку идут только с сервера
- Проверка подписи webhook (`Token`) — обязательна, встроена в библиотеку

## Проверка подписи webhook

Т-Банк отправляет уведомления с полем `Token`. Библиотека вычисляет SHA-256 от:
- Отсортированных по алфавиту корневых параметров (без Token)
- К которым добавлен `Password` (Secret Key)

Совпадение означает, что уведомление действительно от Т-Банка.
