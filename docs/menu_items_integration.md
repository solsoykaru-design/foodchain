# Интеграция раздела «Элементы меню» (YUMA-style)

## Сделанные изменения

### 1. SQL миграция (`server/migrations/013_menu_items_enhanced.sql`)
Добавляет в таблицу `dishes` колонки:
- `unit` (TEXT DEFAULT 'г') — единица измерения
- `netto` (REAL DEFAULT 0) — выход нетто
- `is_active` (INTEGER DEFAULT 1) — активность
- `tenant_id` (INTEGER DEFAULT 0) — мультитенантность
- Индексы для `category_id`, `type`, `barcode`, `article`, `tenant_id`

### 2. Сервер (`server/index.js`)
- **`GET /api/menu-items`** — расширена сортировка: добавлены `barcode`, `article`, `unit`, `netto`, `type`, `is_available`, `markup`. Добавлена специальная обработка сортировки по `markup` (вычисляемое поле).
- **`POST /api/dishes`** — добавлена поддержка полей `barcode`, `article`, `type`, `unit`, `netto`, `cost`, `branch_id`, `tech_card_id`.
- **`PUT /api/dishes/:id`** — добавлена поддержка обновления полей `barcode`, `article`, `type`, `unit`, `netto`, `cost`, `branch_id`, `tech_card_id`.
- **Таблица `dishes`** — расширено CREATE TABLE всеми новыми колонками.

### 3. Фронтенд

#### `src/admin/MenuItemsList.tsx` — полный редизайн
- **Верхняя панель фильтров**: Категория, Магазин, Техкарта (Все/Есть/Нет), Тип (Все/Товар/Услуга), Поиск (по названию, артикулу, штрихкоду)
- **Кнопки действий**: Обновить, Добавить, Удалить, Копировать, Печать, Порядок вывода, Импорт
- **Быстрые фильтры**: Чекбокс «Все» (выбор всех строк), кнопки «Товар» (фильтр по типу) и «Себ-ть» (показать/скрыть колонку себестоимости)
- **Таблица** с 13 колонками: Название, Изображение, Штрихкод, Артикул, Выход (Нетто), Ед. изм., Категория, Тип, Активный (галочка/крестик), Цена, Себестоимость, Наценка, Техкарта
- **Сортировка** по всем колонкам (кроме изображения, категории, техкарты)
- **Пагинация**: Первая, ←, номера страниц, →, Последняя; выбор 20/50/100 элементов; «Страница X из Y»
- **Клик по строке** открывает карточку редактирования

#### `src/admin/MenuItemCard.tsx` — карточка редактирования
- Полноэкранное модальное окно
- Редактирование всех полей: название, цена, себестоимость, артикул, штрихкод, выход (нетто), ед. изм., категория, тип (товар/услуга), активность
- Загрузка изображения (base64 -> upload)
- Отображение вычисляемой наценки
- Создание и редактирование

## Интеграция в существующий раздел «Меню»

### Шаг 1. Применить миграцию
```bash
cd server
node -e "
const db = require('better-sqlite3')('./foodchain.db');
const fs = require('fs');
const sql = fs.readFileSync('./migrations/013_menu_items_enhanced.sql', 'utf8');
db.exec(sql);
console.log('Миграция применена');
"
```

### Шаг 2. Перезапустить сервер
```bash
cd server
node index.js
```

### Шаг 3. Проверить навигацию

### Шаг 4. API эндпоинты

#### GET /api/menu-items
Параметры:
| Параметр | Тип | Описание |
|----------|-----|----------|
| category_id | number | Фильтр по категории |
| store_id | number | Фильтр по магазину/филиалу |
| tech_card_filter | 'all' \| 'yes' \| 'no' | Наличие техкарты |
| type | 'all' \| 'goods' \| 'service' | Тип элемента |
| search | string | Поиск по названию, артикулу, штрихкоду |
| sort_by | string | Поле сортировки |
| sort_order | 'asc' \| 'desc' | Направление |
| page | number | Номер страницы |
| limit | number | Элементов на странице (max 200) |

Ответ:
```json
{
  "items": [{ "id": 1, "name": "...", "categoryName": "...", ... }],
  "total": 100,
  "page": 1,
  "limit": 20,
  "totalPages": 5
}
```

#### POST /api/dishes — создание
Тело: `{ name, price, barcode?, article?, weight?, netto?, unit?, category_id?, type?, is_available?, image_url?, cost?, branch_id?, tech_card_id? }`

#### PUT /api/dishes/:id — обновление
Тело: те же поля, что и POST

### Шаг 5. Дополнительные возможности
- **Магазин**: фильтр по магазинам требует эндпоинт `GET /api/branches` или `GET /api/stores`. Сейчас выпадающий список пуст. Добавьте загрузку магазинов в `useEffect`.
- **Импорт**: кнопка открывает диалог выбора файла (.csv/.xlsx). Обработка файла — заглушка.
- **Массовые действия**: чекбокс «Все» выделяет строки. Можно добавить массовое удаление/активацию через кнопки на тулбаре.
- **Техкарты**: при клике на «Есть» в колонке Техкарта открывается информация о техкарте. Можно заменить `alert` на переход к карточке техкарты.
