# Сравнение back-office FoodChain с конкурентами

Базовая матрица функционала back-office/POS для ресторанного бизнеса.  
Легенда:

- `●` — функция присутствует и развита
- `○` — функция есть, но ограничена / через интеграции
- `-` — нет или неактуально для продукта
- `✅` — реализовано в FoodChain
- `🔄` — частично / заглушки без реальных ключей
- `❌` — не реализовано

## Конкуренты

1. **iiko** — полноценный ресторанный back-office (склад, производство, HR, финансы).
2. **R-Keeper (UCS)** — классический back-office для сетей, сильный финансовый блок.
3. **Poster** — облачный POS + back-office, акцент на простоту.
4. **1С:Ресторан** — учёт, бухгалтерия, производство, интеграция с 1С.
5. **Toast** — американский POS-экосистема с inventory, payroll, аналитикой.
6. **Lightspeed Restaurant (K-series)** — POS + e-commerce + inventory + аналитика.
7. **Jowi** — облачный back-office для кухонь/кофеен, упор на склад и техкарты.

## Основные пробелы FoodChain

1. **Триггерные маркетинговые кампании** — нет конструктора автоматических рассылок (дни рождения, бездействие 30 дней).
2. **RFM-сегментация** — нет анализа Recency/Frequency/Monetary.
3. **Расчёт зарплаты** — есть смены, но нет полноценного расчёта оклада + часов + премий.
4. **Бухгалтерский учёт** — экспорт в 1С заглушен без реальных ключей; нет ОСВ/баланса внутри системы.
5. **Мобильное приложение управляющего** — есть веб-админка, нет нативного приложения.
6. **Маркетплейс приложений / расширений** — есть Extensions SDK, но нет магазина.
7. **Честный знак / ЕГАИС** — есть страница, но требует интеграции с гос. API.

## Рекомендуемый порядок доработки back-office

1. ✅ Склад: инвентаризация, заказы поставщикам, производство — **готово**.
2. Маркетинг: триггерные кампании + RFM.
3. HR: полноценный расчёт зарплаты.
4. Финансы: внутренний баланс и P&L по филиалам.
5. Мобильное приложение управляющего (PWA / Capacitor).
6. Маркетплейс расширений.
7. Честный знак / ЕГАИС.
